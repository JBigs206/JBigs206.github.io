Just Salt It

Just Salt It is a recipe and lifestyle blog website that provides food recipes as well as tips to improve your quality of life and home. While the content on the site is free for everyone to read and learn, members of Just Salt It are also provided with weekly newsletters and access to messaging with the website's owner.

To display the content available to visitors and members of the Just Salt It, we set up each content category page to show the thumbnail and title of the individual posts. Using the Laravel framework, we are able to loop through the returned JSON data from our MySQL table where it is stored.

In order to get the data from the table into the controller to be displayed in the view, we have to distinguish what data we want to access in our model.

Once we have our queries defined in our model, we can then return that data in our controller by assigning a variable to the function in the Content model we are wanting to access and we access the function by referencing the Content class. We can see this being done with the $links variable which creates an array of the data we want and then pass it into the view.

Then in the view, we loop through the $links array using Laravelâ€™s foreach directive to display the thumbnail images and titles of the posts we want to show. We are also able to use the id that is returned for each row to be able to link to the individual post in their own separate view. Using an anchor tag, we can specify the route we want the member to follow with the href attribute and pass the id of the post into the link.

To pass the post id into the controller for the individual post, we have to allow for the controller to accept variables in the Routes file. We distinguish that the route is a GET route and are wanting to activate the index function in the controller we are using.

We then pass the $id variable into the index function in the controller so we can get the specific post the member is navigating to. By setting it up this way, we are able just one controller and one view for each of the three post categories rather than having both a unique controller and view for every single post throughout the site.
