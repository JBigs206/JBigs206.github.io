Emergency Contact Link

Emergency Contact Link is a paid membership hybrid app and website which allows people who experience an emergency to send a notification to specific contacts. Emergency contacts can either be medical or non-medical which determines the type of notification they receive in the event of an emergency. Emergencies can be triggered by a member from within their account or by someone else by navigating to the designated emergency trigger page on the website.

On the emergency trigger page, someone can enter in the victim’s information, along with their information to search for the victim’s membership and initiate the notifications to send to the member’s contacts. Both the victim and aid’s info is collected in the same form to speed up the process of searching for a member and getting the notifications sent.

To achieve this we first need to collect the data in a form and submit it to our api. We do this by specifying that we want to post the data from our inputs to the “emergencyForm” formGroup we have set up in our controller. The inputs then use “formControlName” to specify the names of the data we are sending and submit the data to the “getKickoffInfo()” function.

For the “getKickoffInfo()” function in the controller, we first check if the form we submitted is valid. If it is not valid, we show an error to the user that a required field from the form has been left out and needs to be filled in. If the form is valid then we send the data we collected to our api to check to see if there is a member on file with the corresponding information. If data is returned back from our api, we then push that data to an array we created and repeat the data to the user on the web view. If no data is returned, we show a message to the user that nothing was returned.

To set up returning member info in our Laravel api, we pass our data to the Angular auth service that is pointing to the corresponding Laravel route. Our angular service function communicates to the Laravel back end that it wants to pass the data to the “friendInfo” Laravel route in the “ekickoff” route group. Laravel then passes the data to the “friendInfo()” function in the “kickoffController”.

The “friendInfo()” function first checks to see if the required data it needs to find the correct user is present. If it is not, the function will return a status of “fail” back to the angular controller. If it does pass the validation, then the Laravel controller can search through the database for the member and return the results back to the Angular controller.

If the member that is returned is correct, the user can click the “Trigger Emergency” button which will then send the member’s info along with the aids info from the original form to the “kickoffFriend()” function. This function passes the members id along with the information of the aid to the “kickoffFriendForm” and sends it to our api.

Similarly to how we passed data for finding the member info, we pass the data of the member along with the sender’s info they provided to the Angular auth service. We let the service know that we want to send the data to Laravel route of “friend” in the “ekickoff” route group. In the corresponding Laravel controller, we first check the validation like before and then get the member by searching with the passed in member id. Next, we want to check to make sure the event isn’t being triggered multiple times back to back. If this member has already had an emergency event triggered for them within the last five minutes, it will return a failed result back to the Angular controller letting the user know that an event is already in progress. If this check is passed, we can then post to the emergency event table we have in the database to keep track of events that have been triggered. We then set up a SMS text message notification to be sent to the member the emergency is for, allowing them to cancel the emergency before sending notifications to their emergency contacts. If the member does not respond to the text message within the time frame they set during their registration process, their contacts will then be notified.
