Old EKG

Old EKG is a medical app which allows its users to take or upload pictures of their EKG (electrocardiogram) to their app. This is used for people who find it beneficial to have a secure and handy place to keep records of their EKG’s if they find a need to access them quickly and frequently.

Because this app is storing sensitive medical information of its users, there was a need to have access to the app password protected. To accomplish this, we set up a pin passcode for users to set up and use when accessing the app.

The way we are setting and storing the passcode the user creates, or re-enters after returning to the app, is to pass the numbers the user enters into a function we created. This add() function takes the number value that is passed into it and then goes through a series of checks to determine if the user gets redirected into the app.

In the add function, we first need to point the user's phone’s local storage to either see if a ‘Securitypin’ is already set or set one once the user creates a passcode. Once inside the local storage, we want to set the passcode variable to the values that are being passed in by the user. We then have a int variable, which is initially set to zero, that we are using to determine what position we are going to set the number the user enters in the passcode. If the int variable is zero, then we set the number the user enters to the codeone variable and increment the int variable by one so that the next entry will have the int variable read one and we continue to do this until the user has entered four numbers.

We also want to simultaneously check to see if a user already has a passcode set when they are entering in digits for the passcode. With the ngOnInit() function, which gets called on the page load, we check if a user has a passcode already and then set the variable userPassCode to that passcode. This allows us to perform the check in the add() function which is seeing if userPassCode length is greater than 0. If the length is in fact greater than 0, then that means the user already has a passcode. So if we are able to get inside that check, we then want to check if the user has entered four digits. If the user has entered four digits, then check if the passcode they entered is equal to the pass code which is already stored. If this check succeeds, then we allow the user into the app. But if this check fails, then the user is prompted on the screen that the code is incorrect.

Next, if the check to see if the user has a passcode set fails, we want to see if the passcode length the user has entered has reached 4 digits so we can set a new passcode for the user. We then are doing a check to determine when to prompt the user to re-enter their created passcode so that they can confirm it before being navigated into the app. If the newPinCount variable is equal to zero, then we direct the user to confirm their pin by entering it again. We also set the finalPin variable to equal the values of all the individual digit variable we set in sequential order. Then we set the individual digit variable to null, so the user can re-enter the code, and add one to the newPinCount variable so that it is equal to one. When the user then re-enters their pin, they will pass the first check of newPinCount being greater than zero so we can then allow them into the app and set the local storage ‘Securitypin’ to their entered pincode. They will then be able to use this pincode for future access to the app and not have to reset their pin each time.
